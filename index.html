Introducción al Desarrollo de Interfaces en Android

En esta documentación se explicará el proceso completo para desarrollar interfaces de usuario en Android Studio, incluyendo:

  • Creación y diseño de vistas (layouts)
  • Implementación de menús de navegación
  • Adición de animaciones y transiciones
  • Configuración de archivos de navegación

Requisitos previos

  • Android Studio instalado (versión 2022.3.1 o superior)
  • Conocimientos básicos de XML y Kotlin/Java
  • Proyecto Android configurado con Material Components

Diseño de Vistas en Android Studio

El diseño de interfaces en Android se realiza mediante archivos XML que definen la estructura visual de cada pantalla. Android Studio proporciona dos formas de trabajar:

Editor Visual (Design)

Permite arrastrar y soltar componentes visuales y configurar sus propiedades mediante una interfaz gráfica.

Editor de Código (Text)

Permite editar directamente el XML para un control preciso sobre el layout y sus propiedades.

Ejemplo práctico: Crear un layout básico

1. Crear nuevo archivo XML

En el directorio res/layout/, crear un nuevo archivo XML (ej: activity_main.xml).

Creación de un nuevo archivo de layout en Android Studio

2. Definir la estructura base

Usar ConstraintLayout como contenedor principal:


<!-- activity_main.xml -->
<androidx.constraintlayout.widget.ConstraintLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <!-- Aquí irán los componentes de la interfaz -->

</androidx.constraintlayout.widget.ConstraintLayout>

3. Añadir componentes básicos

Ejemplo de TextView y Button con ConstraintLayout:


<TextView
    android:id="@+id/title"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Bienvenido"
    android:textSize="24sp"
    app:layout_constraintTop_toTopOf="parent"
    app:layout_constraintStart_toStartOf="parent"
    app:layout_constraintEnd_toEndOf="parent"
    android:layout_marginTop="32dp" />

<Button
    android:id="@+id/actionButton"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Continuar"
    app:layout_constraintTop_toBottomOf="@id/title"
    app:layout_constraintStart_toStartOf="parent"
    app:layout_constraintEnd_toEndOf="parent"
    android:layout_marginTop="24dp" />
Vista previa del layout en el editor de Android Studio

Consejos para diseño de vistas

  • Usar ConstraintLayout para diseños responsivos
  • Definir dimensiones en dp (density-independent pixels)
  • Usar sp para tamaños de texto
  • Mantener márgenes consistentes (8dp, 16dp, 24dp, etc.)
  • Extraer strings a res/values/strings.xml

Creación de Menú de Navegación

La Navigation Component de Android Jetpack simplifica la implementación de navegación en tu aplicación. Proporciona una forma consistente y predecible de manejar la navegación entre destinos.

Pasos para implementar un menú de navegación

1. Agregar dependencias

En build.gradle (Module: app):


dependencies {
    // Navigation Component
    implementation "androidx.navigation:navigation-fragment-ktx:2.5.3"
    implementation "androidx.navigation:navigation-ui-ktx:2.5.3"
    
    // Material Components (para BottomNavigationView)
    implementation "com.google.android.material:material:1.8.0"
}

2. Crear el gráfico de navegación

Click derecho en res → New → Android Resource File → Tipo: Navigation

Creación de un nuevo archivo de navegación en Android Studio

3. Definir destinos en nav_graph.xml

Añadir fragmentos/actividades como destinos en el editor de navegación:

Editor visual del gráfico de navegación en Android Studio

4. Implementar BottomNavigationView

Añadir en el layout principal:


<com.google.android.material.bottomnavigation.BottomNavigationView
    android:id="@+id/bottom_nav"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    app:menu="@menu/bottom_nav_menu"
    app:layout_constraintBottom_toBottomOf="parent"
    app:layout_constraintStart_toStartOf="parent"
    app:layout_constraintEnd_toEndOf="parent" />

Crear el archivo de menú (res/menu/bottom_nav_menu.xml):


<menu xmlns:android="http://schemas.android.com/apk/res/android">
    <item
        android:id="@+id/homeFragment"
        android:icon="@drawable/ic_home"
        android:title="Inicio" />
    <item
        android:id="@+id/searchFragment"
        android:icon="@drawable/ic_search"
        android:title="Buscar" />
    <item
        android:id="@+id/profileFragment"
        android:icon="@drawable/ic_profile"
        android:title="Perfil" />
</menu>

5. Configurar NavController en la Activity

En el código Kotlin/Java de la actividad principal:


class MainActivity : AppCompatActivity() {
    
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        
        // Configurar Navigation
        val navController = findNavController(R.id.nav_host_fragment)
        val bottomNav = findViewById<BottomNavigationView>(R.id.bottom_nav)
        
        // Conectar BottomNavigationView con NavController
        bottomNav.setupWithNavController(navController)
    }
}
Menú de navegación inferior funcionando en la aplicación

Tipos de menús de navegación

Bottom Navigation

Ideal para 3-5 destinos principales. Usa BottomNavigationView.

Navigation Drawer

Para muchas opciones. Usa DrawerLayout y NavigationView.

Tabs

Para contenido relacionado. Usa TabLayout con ViewPager2.

Implementación de Animaciones

Las animaciones mejoran la experiencia de usuario haciendo que las transiciones entre estados sean más fluidas y agradables. Android ofrece varios tipos de animaciones:

Property Animation

Animar propiedades de objetos (alpha, traslación, escala, rotación).

Transiciones

Cambios entre actividades o fragmentos con efectos visuales.

Shared Elements

Elementos compartidos entre pantallas con animación.

Ejemplo: Animación de fade in

1. Crear archivo de animación

En res/anim/fade_in.xml:


<?xml version="1.0" encoding="utf-8"?>
<set xmlns:android="http://schemas.android.com/apk/res/android"
    android:interpolator="@android:anim/decelerate_interpolator">
    
    <alpha
        android:fromAlpha="0.0"
        android:toAlpha="1.0"
        android:duration="500" />
</set>
Creación de un nuevo archivo de animación en Android Studio

2. Aplicar la animación en código


val fadeIn = AnimationUtils.loadAnimation(this, R.anim.fade_in)
myView.startAnimation(fadeIn)

Ejemplo: Transición entre actividades

1. Definir transiciones

En res/transition/slide_right.xml:


<?xml version="1.0" encoding="utf-8"?>
<slide xmlns:android="http://schemas.android.com/apk/res/android"
    android:slideEdge="right"
    android:duration="300" />

2. Configurar transiciones en las actividades


// En la actividad origen
val intent = Intent(this, DetailActivity::class.java)
startActivity(intent)
overridePendingTransition(R.anim.slide_in_right, R.anim.slide_out_left)

// En la actividad destino (al cerrar)
override fun finish() {
    super.finish()
    overridePendingTransition(R.anim.slide_in_left, R.anim.slide_out_right)
}
Transición personalizada entre actividades

Ejemplo: Animación de elementos compartidos

1. Definir transición de elementos compartidos

En res/transition/shared_element.xml:


<?xml version="1.0" encoding="utf-8"?>
<transitionSet xmlns:android="http://schemas.android.com/apk/res/android">
    <changeBounds />
    <changeTransform />
    <changeClipBounds />
    <changeImageTransform />
</transitionSet>

2. Configurar en ambas actividades


// En el estilo de la aplicación
<item name="android:windowSharedElementsUseOverlay">false</item>
<item name="android:windowContentTransitions">true</item>

// En la actividad origen
val intent = Intent(this, DetailActivity::class.java)
val options = ActivityOptions.makeSceneTransitionAnimation(
    this,
    Pair(myImageView, "imageTransition")
)
startActivity(intent, options.toBundle())

// En la actividad destino
class DetailActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_detail)
        
        // Asignar transition name al elemento compartido
        detailImageView.transitionName = "imageTransition"
        
        // Posponer transición hasta que la imagen esté cargada
        postponeEnterTransition()
        detailImageView.doOnPreDraw {
            startPostponedEnterTransition()
        }
    }
}
Animación de elementos compartidos entre actividades

Buenas prácticas para animaciones

  • Mantener las animaciones breves (200-500ms)
  • Usar interpoladores naturales (decelerate, accelerate)
  • Evitar animaciones simultáneas complejas
  • Probar en dispositivos de gama baja
  • Considerar desactivar animaciones cuando la batería esté baja

Archivos de Navegación (Navigation)

Los archivos de navegación (navigation graphs) son archivos XML que definen todos los destinos posibles en tu aplicación y las conexiones entre ellos. Centralizan toda la lógica de navegación en un solo lugar.

Estructura de un archivo de navegación

Visualización del gráfico de navegación en Android Studio

Componentes principales

Destinos

Puntos de entrada en tu app (Activities, Fragments, etc.)

Acciones

Conexiones lógicas entre destinos

Argumentos

Datos que se pasan entre destinos

Ejemplo completo de nav_graph.xml


<?xml version="1.0" encoding="utf-8"?>
<navigation xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/nav_graph"
    app:startDestination="@id/homeFragment">

    <!-- Fragmento Home -->
    <fragment
        android:id="@+id/homeFragment"
        android:name="com.example.app.ui.home.HomeFragment"
        android:label="Inicio"
        tools:layout="@layout/fragment_home">
        
        <!-- Acción hacia el fragmento de detalles -->
        <action
            android:id="@+id/action_home_to_detail"
            app:destination="@id/detailFragment" />
    </fragment>

    <!-- Fragmento de Detalles -->
    <fragment
        android:id="@+id/detailFragment"
        android:name="com.example.app.ui.detail.DetailFragment"
        android:label="Detalles"
        tools:layout="@layout/fragment_detail">
        
        <!-- Argumento recibido -->
        <argument
            android:name="itemId"
            app:argType="integer" />
        
        <!-- Acción de regreso al home -->
        <action
            android:id="@+id/action_detail_to_home"
            app:destination="@id/homeFragment"
            app:popUpTo="@id/homeFragment"
            app:popUpToInclusive="false" />
    </fragment>

    <!-- Actividad de Login -->
    <activity
        android:id="@+id/loginActivity"
        android:name="com.example.app.ui.login.LoginActivity"
        android:label="Login"
        tools:layout="@layout/activity_login" />

</navigation>

Uso del NavController

Navegación básica entre destinos


// Navegar usando una acción definida en el grafo
findNavController().navigate(R.id.action_home_to_detail)

// Navegar directamente a un destino
findNavController().navigate(R.id.detailFragment)

// Navegar con argumentos
val bundle = bundleOf("itemId" to 123)
findNavController().navigate(R.id.action_home_to_detail, bundle)

// Navegar usando Safe Args (recomendado)
val direction = HomeFragmentDirections.actionHomeToDetail(itemId = 123)
findNavController().navigate(direction)

Configuración del NavHost

En el layout de la actividad principal:


<androidx.fragment.app.FragmentContainerView
    android:id="@+id/nav_host_fragment"
    android:name="androidx.navigation.fragment.NavHostFragment"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    app:defaultNavHost="true"
    app:navGraph="@navigation/nav_graph" />

Mejores prácticas para archivos de navegación

  • Usar Safe Args para pasar datos entre destinos de forma segura
  • Mantener el grafo de navegación modularizado
  • Definir un destino de inicio claro
  • Usar deep links para navegación desde notificaciones
  • Probar todos los flujos de navegación
